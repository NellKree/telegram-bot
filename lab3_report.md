# Отчёт по лабораторной работе 3
## «Запуск бота для реального использования»

---

## 1. Описание деплоя

**Выбранный способ:** облачный хостинг — **Render (Background Worker)**

**Почему Render:**
- Бесплатный тариф без ограничения по времени работы для воркеров
- Простое подключение GitHub-репозитория
- Автоматический редеплой при пуше в main
- Поддержка переменных окружения через UI

**Ссылка на бота:** `@itmoftmi_ied_bot` в Telegram

---

## 2. Процесс деплоя

**Подготовка проекта:**

Созданы необходимые файлы конфигурации:
- `requirements.txt` — зависимости проекта
- `.gitignore` — исключение `.env`, кэша, временных файлов
- `render.yaml` — конфигурация для Render
- `.python-version` — фиксация версии Python 3.9.6

**Загрузка на GitHub:**
```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/NellKree/telegram-bot.git
git push -u origin main
```

**Настройка на Render:**
1. Создан Background Worker (не Web Service — у воркера нет портов)
2. Build Command: `pip install -r requirements.txt`
3. Start Command: `python bot.py`
4. Добавлены переменные окружения: `BOT_TOKEN`, `GEMINI_API_KEY`

## 3. Проблемы при деплое и решения

| Проблема | Причина | Решение |
|---|---|---|
| Railway недоступен | Платформа приостановила деплои для триал-пользователей | Переключился на Render |
| `apt-get: Permission denied` | Нет прав root на Render | Убрал `apt-get`, ffmpeg уже предустановлен |
| `requirements.txt not found` | В имени файла был лишний пробел после `txt` | Переименовал через `mv "requirements.txt " requirements.txt` |
| `TelegramConflictError` | Бот был одновременно запущен локально и на Render | Остановил локальный процесс |
| `No open ports detected` | Render ожидал веб-сервис, а не воркер | Пересоздал сервис как Background Worker |
| Python 3.14 вместо 3.9 | Render использовал версию по умолчанию | Добавил файл `.python-version` с `3.9.6` |

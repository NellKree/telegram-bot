# Отчёт по лабораторной работе 2
## «Подключение бота к данным»

---

## 1. Описание интеграции

Выбранный тип интеграции — **внешние API и файловое хранилище**. Бот интегрирован с двумя источниками данных:

1. **Google Drive API (через gdown)** — скачивание аудио/видео файлов по публичной ссылке
2. **Gemini API (Google)** — генерация структурированного summary на основе транскрипта

Выбор обусловлен практической ценностью: пользователь работает с реальными файлами из облака, а не с тестовыми данными.

---

## 2. Промпт для LLM

```
Улучши Telegram-бота, добавив интеграцию с Google Drive и Gemini API.

Текущий функционал: бот принимает текстовые сообщения.

Новый функционал:
- Принимать ссылку Google Drive, скачивать файл через gdown
- Проверять размер файла (лимит 20 ГБ) и формат (mp3/wav/ogg/mp4/mov/avi/mkv)
- После транскрипции отправлять файл transcript.txt пользователю
- Делать запрос к Gemini API через requests (не через SDK)
- Ответ Gemini форматировать: summary, ключевые решения, action items

API endpoint: https://generativelanguage.googleapis.com/v1beta/models/gemini-flash-latest:generateContent
```

---

## 3. Реализация

**Скачивание файла (`drive_downloader.py`):**
Используется `gdown` с параметром `fuzzy=True` для поддержки разных форматов ссылок Google Drive. После скачивания определяется расширение файла через `mimetypes`, если не определяется — перебираются возможные форматы.

**Запрос к Gemini API (`gemini.py`):**
Запрос отправляется через `requests.post` с API-ключом в заголовке `X-goog-api-key`. Промпт инструктирует модель сделать структурированное резюме: краткое summary, ключевые решения, action items, важные моменты. Реализована обработка ошибок API и неожиданных форматов ответа.

**Ключевой фрагмент запроса к API:**
```python
response = requests.post(url, headers=headers, json=payload, timeout=60)
data = response.json()
return data["candidates"][0]["content"]["parts"][0]["text"]
```

## 4. Тестирование

Протестированы форматы: `.mp3`, `.mp4`, `.ogg`. Бот корректно скачивает файлы, транскрибирует и возвращает summary. Обработка ошибок проверена на недоступной ссылке и неподдерживаемом формате.

*Скриншоты работы добавлены в приложении.*

---

## 5. Трудности и решения

| Проблема | Решение |
|---|---|
| gdown не определял тип файла | Добавлен перебор расширений после скачивания |
| Gemini иногда возвращает ответ без `candidates` | Добавлена проверка и информативное сообщение об ошибке |
| Большие файлы блокируют event loop | Обёртка `asyncio.to_thread()` для синхронных операций |

---

## 6. Выводы

Интеграция с реальными API значительно увеличила ценность бота. Самой сложной частью оказалась обработка edge cases при работе с файлами. В будущем можно добавить поддержку ссылок из других облачных хранилищ (Яндекс Диск, Dropbox).

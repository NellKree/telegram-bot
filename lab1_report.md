# Отчёт по лабораторной работе 1
## «Создание первого Telegram-бота без программирования»

---

## 1. Описание задачи

Бот решает задачу автоматической транскрипции аудио и видео материалов. Идея возникла из реальной потребности: часто после встреч, лекций или интервью остаются записи, которые сложно быстро обработать вручную. Бот принимает ссылку на файл в Google Drive и возвращает текстовую расшифровку.

**Функционал бота:**
- Приём ссылки Google Drive на аудио/видео файл
- Скачивание файла
- Транскрипция речи в текст с помощью Vosk
- Генерация структурированного summary через Gemini API
- Отправка результата пользователю в виде текстового файла

---

## 2. Промпт для LLM

Для генерации кода использовался Claude. Исходный промпт:

```
Создай Telegram-бота на Python с использованием библиотеки aiogram 3.
Функционал:
- Бот принимает ссылку Google Drive на аудио или видео файл
- Скачивает файл с помощью gdown
- Транскрибирует речь в текст с помощью Vosk (русская модель vosk-model-small-ru-0.22)
- Отправляет транскрипт пользователю как текстовый файл
- Предлагает сделать summary через кнопку
- Summary генерируется через Gemini API (через requests, без SDK)
Требования: хранить токен в .env, обработка ошибок, поддержка форматов mp3/wav/ogg/mp4/mov
```

Итерации: уточнялась обработка ошибок при скачивании, добавлена проверка размера файла и формата.

---

## 3. Стек технологий

- **aiogram 3.7** — асинхронная библиотека для Telegram Bot API, выбрана за современный синтаксис и поддержку inline-кнопок
- **gdown** — скачивание файлов с Google Drive по публичной ссылке
- **vosk** — офлайн распознавание речи, модель `vosk-model-small-ru-0.22`
- **ffmpeg** — конвертация аудио/видео в формат для Vosk (16kHz, mono, PCM)
- **requests** — HTTP-запросы к Gemini API
- **python-dotenv** — хранение токенов в переменных окружения

## 5. Трудности и решения

| Проблема | Решение |
|---|---|
| Vosk не определял расширение скачанного файла | Добавлена логика перебора расширений в `drive_downloader.py` |
| ffmpeg не был установлен в системе | Установлен отдельно через brew, добавлен в конфиг деплоя |
| Gemini API возвращал нестандартный ответ | Добавлена защита с проверкой наличия ключа `candidates` в ответе |
